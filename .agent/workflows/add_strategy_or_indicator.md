# Workflow: Adding a New Indicator or Strategy

This workflow outlines the exact steps the AI must take when asked to augment the trading system with new technical indicators or trading strategies.

## 1. Strategy Reuse & Modularity (Check First)
**CRITICAL**: Before creating a new strategy class, you MUST verify if the logic can be handled by an existing parameterized strategy.
- Example: "Exit when price drops below SMA(20)" is NOT a new strategy. It is simply `SimpleMAStrategy(short_window=1, long_window=20)`.
- **Inversion**: If the user asks for contrarian logic ("Buy when price drops below SMA(20)"), use the `invert=True` parameter on existing strategies, which multiplies the signal by -1.

## 2. Centralized Indicator Rule
**CRITICAL**: Strategies must NEVER compute indicators internally using Pandas math. All indicators must be computed upstream in `IndicatorEngine`.

If the user requests an indicator that isn't currently supported, you must first add it:
1. Open `trading_on_tcbs_api/stock_system_v2/core/indicator_engine.py`
2. Add the requested parameter to the dictionary in `get_default_config()` (e.g. `"atr": [14]`).
3. Add a new `if` block in `append_indicators(self, df)` that maps the config to the `pandas-ta` library using `df.ta.{indicator}(...)`. For example:
```python
if "atr" in self.config:
    for length in self.config["atr"]:
        data.ta.atr(length=length, append=True)
```
4. Remember that `pandas-ta` generates uppercase columns by default, but the `IndicatorEngine` lowercases all columns before returning. The column will be e.g., `atr_14`.

## 3. Implement the Strategy
If writing a new strategy, it must follow the `SignalStrategy` interface:
1. Create a new file in `trading_on_tcbs_api/stock_system_v2/strategies/`.
2. **REQUIRED DOCS**: Define a `self.name` and concise `self.description` attribute in the `__init__` constructor summarizing EXACTLY what conditions trigger the signal (e.g., `self.description = "BUY when RSI < 30"`). This will be formatted by the base class's `get_brief()` and printed by the scanner engine for transparency.
3. **DYNAMIC COLUMNS**: NEVER hardcode indicator lengths into column names (e.g., DO NOT USE `'sma_20'` or `'%_from_sma20'`). Always use dynamic f-strings tied to the strategy's initialized parameters (e.g., `f'sma_{self.sma_window}'` and `f'%_from_sma{self.sma_window}'`). This prevents catastrophic key errors when parameters are changed.
4. Ensure the `generate_signals` method reads the pre-calculated, lowercased column generated by `pandas-ta`.
5. Throw a `ValueError` if the column doesn't exist (this protects against config drift).
6. Do NOT calculate the math directly in the strategy.
7. Set `df['signal'] = 1` for BUY and `-1` for SELL based on DataFrame slicing.

## 4. Wire Up the Strategy
Inject the new strategy into the `CombinedStrategy` in the following execution files to test it:
- `trading_on_tcbs_api/stock_system_v2/scripts/scan_market.py`
- `trading_on_tcbs_api/stock_system_v2/scripts/verify_backtest.py`
- `trading_on_tcbs_api/stock_system_v2/core/auto_trader.py`

Run `verify_backtest.py` to ensure it doesn't crash before finishing the task.
